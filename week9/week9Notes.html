Week 9 Notes: Window Object & API's

Window Object

The window object allows the user to interact with the window that the user is on. There are a few things that it can do. See W3Schools Window Object. Here are a few of the more relevant things that it can do

Alert

alert("This is an alert")

This method allows the user to display an alert on the window when the code gets executed. This is usually used for debugging purposes.

Status

window.status = "Hello World"

This code returns text into the status bar

Local Storage

// Store localStorage.setItem("lastname", "Smith"); // Retrieve document.getElementById("result").innerHTML = localStorage.getItem("lastname");

This method allows the user to save the information locally

Session Storage

// Store sessionStorage.setItem("lastname", "Smith"); // Retrieve document.getElementById("result").innerHTML = sessionStorage.getItem("lastname");

This method allows the user to store information as long as the session is still continuing. The information is discarded once the session ends

API's

an API is a way for a developer to quickly access information from a seperate database. It stands for Application Program Interface. API calls in Javascript using HTTP requests. This requires a person to have an api url an example of this can be found in PokeApi example. A request usually looks along the lines of this.

// Create a request variable and assign a new XMLHttpRequest object to it. var request = new XMLHttpRequest() // Open a new connection, using the GET request on the URL endpoint request.open('GET', 'https://ghibliapi.herokuapp.com/films', true) request.onload = function() { // Begin accessing JSON data here } // Send request request.send()

A forewarning, http requests are asynchronous therefore everything doesn't happen chronologically but it happens close to the same time.
