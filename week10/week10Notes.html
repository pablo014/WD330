Form Validation and Fetch

Form Validation

Form validation is a way for the developer to limit the types of inputs for the user. This allows the user to be able to input usable data for the developer. Examples of this can be found in logins, account creation, and payment options in differnt websites. Whenever a form requires you to fill out a specific piece of information or fill it out in a certain way, this is a form of form validation.

Form validation can use many techniques such as

  • Required
  • Min and Max
  • Regular Expressions
  • Type
  • Pattern

Fetch

Fetch is a quick way for the user to make an API call using an XMLHTTP request. I usually follows this pattern.

fetch('insertUrlHere.dev') .then(response => response.json) .then( data => function() { //add code here console.log(data) } )

Fetch can also take in a second parameter to control different settings

const response = await fetch(url, { method: 'POST', // *GET, POST, PUT, DELETE, etc. mode: 'cors', // no-cors, *cors, same-origin cache: 'no-cache', // *default, no-cache, reload, force-cache, only-if-cached credentials: 'same-origin', // include, *same-origin, omit headers: { 'Content-Type': 'application/json' // 'Content-Type': 'application/x-www-form-urlencoded', }, redirect: 'follow', // manual, *follow, error referrerPolicy: 'no-referrer', // no-referrer, *no-referrer-when-downgrade, origin, origin-when-cross-origin, same-origin, strict-origin, strict-origin-when-cross-origin, unsafe-url body: JSON.stringify(data) // body data type must match "Content-Type" header });

Different settigns that fetch can manipulate are credentials, whether you are pushing data in or pulling it out, etc.
